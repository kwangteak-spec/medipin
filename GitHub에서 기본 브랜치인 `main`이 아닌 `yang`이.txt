GitHub에서 기본 브랜치인 `main`이 아닌 `yang`이라는 이름의 브랜치로 코드를 올리는 방법은 아주 간단합니다. 현재 상황에 따라 크게 두 가지 방법으로 나뉩니다.

---

## 1. 처음부터 `yang` 브랜치를 만들어 올릴 때

로컬 컴퓨터에 아직 브랜치가 없거나, 새로운 브랜치를 만들어서 올리고 싶을 때 사용하는 방법입니다.

1. **브랜치 생성 및 전환:**
터미널(또는 CMD)에서 아래 명령어를 입력합니다.
```bash
git checkout -b yang

```


*(이 명령은 `yang`이라는 브랜치를 만들고 동시에 그 브랜치로 이동합니다.)*
2. **파일 추가 및 커밋:**
```bash
git add .
git commit -m "yang 브랜치에 올리는 첫 메시지"

```


3. **GitHub에 푸시:**
```bash
git push origin yang

```



---

## 2. 기존 브랜치 이름을 `yang`으로 바꿀 때

현재 `main`이나 다른 이름의 브랜치에서 작업 중인데, 이 이름 자체를 `yang`으로 바꿔서 올리고 싶을 때입니다.

1. **현재 브랜치 이름 변경:**
```bash
git branch -m yang

```


2. **GitHub에 푸시:**
```bash
git push -u origin yang

```


*(`-u` 옵션을 쓰면 다음부터는 `git push`만 입력해도 자동으로 `yang` 브랜치로 올라갑니다.)*

---

### 💡 팁: GitHub에서 기본 브랜치 변경하기

만약 앞으로 모든 작업을 `main`이 아닌 `yang`에서 하고 싶다면, GitHub 웹사이트에서도 설정을 바꿔줘야 합니다.

* **GitHub 저장소 접속** > **Settings** > **Branches**
* **Default branch** 항목 옆의 화살표 버튼(Switch)을 눌러 `yang`으로 변경

> **주의:** 이미 GitHub에 `main` 브랜치에 코드가 올라가 있다면, `yang` 브랜치를 새로 푸시했을 때 GitHub 상에는 두 개의 브랜치가 공존하게 됩니다.
